# Contributing to HDA Vault

First, thank you for your interest in improving HDA Vault! To protect the intellectual property and ensure the project's long-term stability, I have established the following contribution guidelines.

## Contributor License Agreement (CLA)

By contributing to HDA Vault, you agree to the following terms:

1. **Copyright Assignment:** You assign all rights, title, and interest in your contributions to **Raj Mitra**.
2. **License to You:** Raj Mitra grants you a non-exclusive, worldwide, royalty-free license to use your own contributions in other projects.
3. **Representation:** You represent that you are the sole author of your contributions and have the legal right to assign the copyright.

## How to Contribute

1. **Fork the Repository:** Create your own copy of the project.
2. **Create a Branch:** Work on your changes in a feature branch.
3. **Open a Pull Request:** Describe your changes clearly and link to any relevant issues.

## Coding Standards

- Follow the existing architectural patterns (e.g., Hive structure, 24-bit color logic).
- Ensure all new features are multi-threaded using Web Workers where applicable.
- Maintain high security standards by using the WebCrypto API.

## Testing

Before submitting a pull request, ensure that your changes have been tested across multiple browsers and that they do not break existing HDA Protocol functionality.

## Questions?

If you have any questions or need clarification, please open an issue or reach out to Raj Mitra.
